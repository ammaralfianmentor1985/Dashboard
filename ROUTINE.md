# Daily Briefing Routine (Claude cloud routine — no API key)

The daily **Market Update Harian Premium** is generated by a **scheduled Claude
cloud routine**, not by GitHub Actions. The routine lives in the owner's
claude.ai account and runs on Anthropic's servers on the normal Claude
subscription — there is **no Anthropic API key and no per-run bill**.

Because everything lives in the cloud (routine → claude.ai account, code →
GitHub, site → Netlify), **no computer needs to be on** at 5AM, and changing or
adding computers requires no setup — just log into claude.ai with the same
account to view or edit the routine.

## Schedule

- **Daily at 04:45 WIB** (Asia/Jakarta, UTC+7) — the finished briefing is
  committed and deployed by roughly 5:00 AM WIB.

## Pipeline

```
Claude cloud routine (04:45 WIB, on subscription)
  → web-searches today's real market news (every point cited)
  → writes narrative.json  (not committed)
  → node scripts/generate-briefing.mjs --narrative narrative.json
      → fetches free live quotes (Stooq / CoinGecko / Frankfurter)
      → renders briefings/market-update-YYYY-MM-DD.html  (ID/EN/MY toggle)
      → prints briefings/market-update-YYYY-MM-DD.pdf    (Playwright Chromium)
      → regenerates briefings-data.js
  → commits those files to main and pushes
  → Netlify auto-deploys → https://stately-pegasus-34152a.netlify.app
```

## Managing the routine

Open **claude.ai → Code → Routines** (any device, any computer, same account)
to view runs, edit the schedule, pause, or delete. If it ever needs to be
recreated, make a new routine on this repository with the schedule above and
the exact prompt below.

## Routine prompt (verbatim)

```
Generate and publish today's "Market Update Harian Premium" for this Dashboard repo.

CONTENT RULES (non-negotiable):
- Report what is ACTUALLY happening in markets, positive or negative — never force one-sided framing for any region.
- Persuasive, confident senior-portfolio-manager tone is fine; every fact must be real.
- EVERY point must cite a real source: outlet name + date, e.g. "CNBC, 9 Jul 2026".
- Never invent, estimate, or interpolate numbers. If a figure cannot be verified, omit it or write "data tidak tersedia".
- Write all narrative text in Bahasa Indonesia (the page's EN/MY toggle translates the UI chrome, not the bullet content).

STEPS:
1. Compute today's date in Asia/Jakarta as YYYY-MM-DD.
2. Research today's markets with web search: US stocks (indices, movers, sectors), forex (USD strength, EUR/USD, USD/IDR, USD/MYR), crypto (BTC, ETH, drivers), commodities (gold, oil, silver), Indonesia (IHSG, rupiah, macro news), Malaysia (KLCI, ringgit, macro news), 2-4 profit opportunities, and a 3-5 point executive summary.
3. Write narrative.json in the repo root (do NOT commit it), exactly this shape:
   {
     "sentimentScore": <0-100, optional>,
     "sentimentLabel": "<short label, optional>",
     "usStocks":    { "badge": "Positif|Netral|Waspada|Negatif", "points": [{ "text": "...", "source": "Outlet, D Mon YYYY" }] },
     "forex":       { same shape },
     "crypto":      { same shape },
     "commodities": { same shape },
     "indonesia":   { same shape },
     "malaysia":    { same shape },
     "opportunities": [{ "market": "...", "icon": "<emoji>", "text": "...", "source": "..." }],
     "summary": ["...", "..."]
   }
   (opportunities max 4, summary max 5)
4. Run:
   npm install
   npx playwright install --with-deps chromium
   node scripts/generate-briefing.mjs --narrative narrative.json
5. Check the generated briefings/market-update-<date>.html for "data tidak tersedia" stat cards. For any missing quote, verify the real value via web search from a reputable source, write market-override.json using the same field shapes as scripts/fetch-market-data.mjs output (include source and url), then re-run:
   node scripts/generate-briefing.mjs --narrative narrative.json --market market-override.json
   Only include values you actually verified — otherwise leave "data tidak tersedia".
6. Verify briefings/market-update-<date>.html and briefings/market-update-<date>.pdf exist and briefings-data.js lists today's entry first.
7. Commit ONLY briefings/market-update-<date>.html, briefings/market-update-<date>.pdf and briefings-data.js to main with message "Daily market briefing <date>" and push. Never commit narrative.json or market-override.json.
8. Netlify auto-deploys from main — nothing else to do. If research, generation, or push fails, stop and report the exact error.
```

## Manual fallbacks

- **From any machine with Node:** write a `narrative.json` (shape above), then
  `npm install && npx playwright install chromium && node scripts/generate-briefing.mjs --narrative narrative.json`,
  commit the outputs to main.
- **GitHub Action** (`.github/workflows/daily-briefing.yml`, "Run workflow"
  button): regenerates a briefing on demand, but its narrative step calls the
  Anthropic API and needs a paid `ANTHROPIC_API_KEY` repository secret — it is
  intentionally **not** scheduled.
